I have done some simple tests on some ruby frameworks - Rails, Sinatra, Padrino and Goliath.

There are testing environment:

  • Ubuntu 12.04
  • CPU Core i5 3450
  • DDR3 8GB 1333 MHz
  • HDD 500 Gb
  • ENV=development
  • MySQL Version: 5.5.35
  • ApacheBench, Version 2.3
  • Ruby version: 2.1.0
  • Rails version: 4.0.2
  • Sinatra: 1.4.4
  • Padrino 0.11.4
  • Goliath 1.0.2

Testing scenarios:

  1. No Views and DB
  2. Views (Slim)
  3. MySQL*
  4. Views (Slim) + MySQL*

* As a DB adapter I've used Sequel (simple load record column from DB)

Testing was done with ApacheBench 2.3 (ab -n 1000 -c 100 http://localhost:PORT/)

Resulting graphs (time in seconds shows how much time was needed for passing test):

Conclusion: We can see that Padrino showed a good result - so you can easily use it (it has MVC structure like rails) for non-large web project. Sinatra was very similar to padrino, Rails was very good too and Goliath showed the worst result. Hope this helps you with choosing the Ruby framework.
